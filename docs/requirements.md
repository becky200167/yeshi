# 需求整理（依据开题报告）

## 1. 业务目标

构建面向城市夜市场景的 WebGIS 可视化平台，实现夜市摊位空间信息的采集、展示、查询与管理，提升管理决策效率与用户消费体验。

## 2. 角色与功能需求

### 2.1 普通用户

- 地图浏览：查看夜市摊位空间分布。
- 信息查询：点击摊位查看类型、经营内容、营业时间等详情。
- 热度可视化：通过热力图查看夜市活跃区域。

### 2.2 商户

- 摊位信息展示：展示位置、经营类别、营业时段。
- 摊位信息维护：提交新增/修改信息。
- 审核流程：商户提交后需管理员审核通过才可对外展示。

### 2.3 管理员

- 数据管理：新增、修改、删除、审核摊位数据。
- 空间分布管理：依据可视化结果进行布局优化。
- 运行维护：服务运行与数据一致性保障。

## 3. 非功能需求

- 可用性：界面直观、操作简单。
- 可扩展性：前后端分离，便于扩展统计分析、用户管理。
- 可维护性：分层结构与模块化实现。
- 性能与稳定性：中小规模数据下流畅运行。
- 安全性：基础入参校验、接口访问约束。

## 4. 技术方案

- 前端：HTML/CSS/JavaScript + Leaflet + HeatLayer。
- 后端：Flask REST API。
- 数据：GeoJSON 风格坐标 + 业务属性；当前采用 JSON 文件，后续可迁移数据库。
- 可视化：点位符号 + 热力图。

## 5. 最小可交付版本（MVP）

- 摊位列表与单点详情接口。
- 前端地图点位展示与弹窗详情。
- 热力图数据接口与前端切换。
- 商户提交接口（待审核）。
- 管理员审核接口（通过后写回主数据）。
